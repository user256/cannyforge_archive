# Developer Hooks

CannyForge Archive exposes a small set of WordPress action and filter hooks so
themes and other plugins can extend its behaviour without modifying core code.

## Filters

### `cannyforge_archive_entries`

Applied to the array of {@see \CannyForge\Archive\Contracts\Archive\ArchiveEntry}
objects immediately before they are rendered.

**Parameters:**
- `$entries` (`ArchiveEntry[]`) — The entries returned by the provider.

**Example:**
```php
add_filter( 'cannyforge_archive_entries', function ( array $entries ): array {
    // Remove entries that lack a featured image.
    return array_filter( $entries, fn ( $e ) => '' !== $e->featured_image_url() );
} );
```

### `cannyforge_archive_theme_css`

Applied to the CSS variable string generated by {@see ThemeCssBuilder}.

**Parameters:**
- `$css` (`string`) — The raw CSS variable block.
- `$theme` (`\CannyForge\Archive\Contracts\Settings\Theme`) — The current theme settings.

**Example:**
```php
add_filter( 'cannyforge_archive_theme_css', function ( string $css, $theme ): string {
    return $css . '.cannyforge-archive{ border-radius: 8px; }';
}, 10, 2 );
```

## Actions

### `cannyforge_archive_before_render`

Fired immediately before the archive HTML is rendered (cache miss only).

**Example:**
```php
add_action( 'cannyforge_archive_before_render', function (): void {
    // Inject tracking pixel, analytics event, etc.
} );
```

### `cannyforge_archive_after_render`

Fired immediately after the archive HTML is rendered (cache miss only).

**Example:**
```php
add_action( 'cannyforge_archive_after_render', function (): void {
    // Log render time, trigger cache warm elsewhere, etc.
} );
```

## Internal actions

### `cannyforge_archive_settings_saved`

Fired inside {@see \CannyForge\Archive\Core\Settings\OptionsSettingsRepository::save()}
after settings have been persisted. Used internally by the cache invalidator to
flush the archive HTML transient when configuration changes.
